# 🏛️ THEME: **“CIVIL WAR ROOM”**

A fusion of:

* **Ancient authority (stone, permanence, decrees)**
* **Modern command center (data, screens, urgency)**

Not playful. Not “startup friendly.”
This thing should feel like:

> “Once an issue is logged, it becomes history. And history doesn’t forget.”

---

# 1. 🎨 VISUAL & CSS IDENTITY

## 🔥 Core Visual Metaphor

* Dashboard = **War Table**
* Issues = **Filed Reports**
* Comments = **Orders / Updates**
* History = **Permanent Record / Archive**

---

## 🧱 Design Language

### Surfaces

* Dark stone textures + matte panels
* Subtle grain, not Pinterest marble nonsense

### Borders

* Thick, deliberate, slightly imperfect lines
* Inspired by carved stone edges

### Elevation

* No soft shadows
* Use **hard light + sharp contrast**

---

## 🎯 CSS STYLE SYSTEM

### Style Mix:

* **Neo-brutalism (structure, boldness)**
* **Low-opacity glass panels (modern overlay)**
* **Grid-heavy layouts (command center feel)**

---

### Example CSS Concepts


# 2. 🎬 THE PRODUCT SYSTEM (THE “MOVIE”)

Stop thinking screens. Think tension.

---

## 🎥 ACT 1: ENTRY

* Login = **Identity Verification**
* Join building = **Entering Territory**

---

## 🎥 ACT 2: WAR TABLE (Dashboard)

* All issues visible

No hiding. No DMs. Public accountability.

---

## 🎥 ACT 3: INCIDENT REPORT

* Title
* Category

* Leaves “official” comment

---

## 🎥 ACT 5: RESOLUTION

* Issue marked resolved

## 🎥 ACT 6: THE FUTURE FIGHT

Tenant:

* Opens archive
* Pulls history

That’s the entire product.
Conflict → Evidence → Power.

# ⚙️ CORE ENGINE (LOGIC)

## Issue System

* Immutable creation timestamp
* Status transitions only by admin
* Severity affects visibility priority

---

## Comment Threads

* Linear timeline
* No deletion (only soft-hide by admin)
* Everything logged

---

## Role System

* Tenant → create/view
* Landlord → control status + building

# 🧠 3. TECH STACK OPTIONS (YOU PICK, STOP OVERTHINKING)

## 🅰️ Firebase

### Pros:

* Easy auth
* Real-time updates
* Fast setup

### Structure:

* Firestore collections:

  * users
  * buildings
  * issues
  * comments

### Logic:

* Security rules enforce:

  * only landlord updates status
  * tenant limited access

---

## 🅱️ Supabase

### Pros:

* SQL (clean relations)
* Row-level security (powerful)

### Tables:

* users
* buildings
* issues
* comments

### Logic:

* RLS policies enforce roles
* Joins for:

  * issues per building
  * comments per issue

# 🧪 6. EDGE CASES (WHERE MOST PROJECTS DIE)

## 🚫 Duplicate Spam

* Same title + same user within 5 mins → warn
* UI: “This looks similar to an existing issue”

---

## 🚫 Unauthorized Actions

* Tenant tries to change status → blocked
* Show:

  > “Restricted command”

---

## 🌐 Network Failure

* Show:

  * “Transmission failed”
* Retry option
* Optimistic UI rollback

---

## 🧠 State Desync

* Always re-fetch after critical updates

---

## 🧱 History Integrity

* No hard deletes
* Audit log for:

  * status changes
  * edits
