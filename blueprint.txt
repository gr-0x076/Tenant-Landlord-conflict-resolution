# 🎬 PROJECT BLUEPRINT: “Tenant War Room”

## 1. 🎯 Problem Statement (what your evaluator actually cares about)

You don’t have a product until this is sharp:

**User:**
Students + young professionals living in PGs / hostels / rented flats

**Problem:**
Communication between tenants and landlords is:

* fragmented (WhatsApp chaos)
* unverifiable (“you never told me”)
* unstructured (no priority, no tracking)

**Impact:**

* delayed repairs
* deposit disputes
* constant friction

**Solution:**
A **transparent issue tracking system for living spaces** where:

* issues are logged
* status is tracked
* history is permanent

This directly satisfies the “real-world non-trivial problem” requirement from your guidelines 
Not a toy. Not a clone. Finally.

# 🎥 2. The “Movie” (User Flow Story)

You wanted a director-level vision. Here:

## Scene 1: Entry

* User signs up
* Chooses role:

  * Tenant
  * Landlord / Warden

## Scene 2: Joining the “Building”

* Tenant enters invite code
* Lands in **Building Dashboard**

## Scene 3: Conflict Begins

* Water not working
* Tenant logs issue:

  * title
  * category
  * severity
  * photo

## Scene 4: Public Exposure

* Issue appears in shared feed
* Other tenants see it
* No hiding anymore

## Scene 5: Authority Responds

* Landlord updates:

  * “In progress”
  * comment: “plumber at 5 PM”

## Scene 6: Resolution

* Issue marked resolved
* History stays forever

## Scene 7: Future Fight

* Tenant leaving → deposit argument
* Opens history
* Pulls receipts
* Wins

That’s your product. Conflict → visibility → accountability.

# 🏗️ 3. Core Features (don’t overbuild like an idiot)

## Feature 1: Issue System (MAIN ENGINE)

* Create issue
* Edit (only creator)
* Delete (optional)
* Status updates (admin only)

## Feature 2: Comment Thread

* Per issue discussion
* timestamped
* no editing (keep it real)

## Feature 3: Role-Based Access

* Tenant:

  * create/view issues
* Landlord:

  * update status
  * manage building

## Feature 4: Building System

* Join via code
* One user → one building (keep it simple)

## Feature 5: Dashboard

* Filters:

  * Open / Closed
  * Category
* Stats:

  * issues this week
  * most common problem

## Must-use React Concepts (from guidelines )

### Core

* useState → forms, filters
* useEffect → fetch issues
* props → pass issue data

### Intermediate

* Context API → user + building
* Routing →

  * /login
  * /dashboard
  * /issue/:id
  * /admin

### Advanced (you want marks?)

* useCallback → avoid rerenders
